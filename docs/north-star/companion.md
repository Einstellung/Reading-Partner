# 伴读宠物（常驻形象）

## 愿景

让 partner 真的有 partner 的样子。一个按钮唤出桌面卡通宠物：可以逗弄它，它也会主动说说话；随时能和它直接开聊，语音为主，文字也行。

它是"不划线直接提问"这个入口的最终形态——不做成工具栏上一个干巴巴的按钮，做成一个活的伴侣。

## 为什么现在不做

全新的大块功能：形象与动画、语音链路（STT/TTS）、主动说话的策略，都不在当前基本盘。且主动说话和主动联想同理——内容来自记忆，记忆没立起来之前的主动是乱说。要等 AI 地基（agent 工具循环、[memory](./memory.md)）就绪后再做。

## 将来做时已知的事实

- 主动说话的内容来源是 memory 的主题快照与蒸馏记忆，依赖 [memory](./memory.md) 先落地。
- 语音链路在零后端约束下走各 provider 的语音 API 或本地模型，需要专门调研。
- 与通话模型（[03](../03-陪读交互-通话模型.md)）的关系：划线触发的通话仍是主场景，宠物是常驻入口，两者共享同一套 AI 地基。
- 用户以 iPad 为主，形态按 iPad 设计。
