# Memory（记忆与主动联想）

## 愿景

挂断时静默蒸馏对话，沉淀进主题 memory；开场注入主题快照，AI 对"你读到哪、卡在哪、信什么"心里有数；主动联想——读这本时自然接上你在那本划过聊过的东西；"看看 AI 记了什么"查看面板；对话里说"你记错了"，AI 自己修正。

## 为什么现在不做

人的联想在记忆里发生，没有 memory 的主动联想没有方向，只是检索结果的堆砌。先把 AI 地基立起来——M6 的 agent 工具循环和"看得见书"（全文、上下文装配、主题内检索），用起来攒了体感再回来做。

## 将来做时已知的事实

- 设计已定稿，全在 [02](../02-AI核心与memory设计.md) 第二部分：结构学 Claude Code（一事一文件、索引+明细、写入端策展），管线学 OpenClaw（transcript 存档 → 挂断蒸馏 → 长期 memory、时间衰减），领域层自己做（证据锚点指回划线、领域 schema、⭐ 权重）。
- 借自 HMS 的三条规矩（2026-07-17，新智元报道的记忆系统；其引擎是 Python 后端 + pgvector，不采用——读取端重活是为"海量无策展日志"设计的，我们写入端策展后用不上）：
  1. 主动回忆：recall 走 agent 工具循环，prompt 里约束回答用户历史问题前先想清楚找什么（哪本书、哪个概念、什么时候），检索后对着问题自查证据够不够，不够换词再搜。
  2. 时间锚定在写入端：蒸馏落笔时把相对时间写成绝对日期，不留"上周"、"最近"。
  3. 矛盾时保留演变：新蒸馏与旧条目冲突（用户从不懂到懂）不抹掉旧状态，更新成演变记录（"曾卡在 X，某日已理清"）——教学要靠"曾经不懂什么"，纵向分析以 transcript 为准（memory 只是派生视图）。
- 蒸馏触发点：挂断为主，上下文将压缩时兜底（[03](../03-陪读交互-通话模型.md)）。
- 检索第一版 BM25 够用，长大后再评估混合检索（memsearch）。
- Memory Adapter 窄接口（retain/recall/listObservations/correct/rebuild），引擎可替换。
