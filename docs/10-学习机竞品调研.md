# 学习机竞品调研

2026-07-17 调研定稿。对照 [09-学习机模式](./09-学习机模式.md) 的五个特征扫了主流学术阅读/AI 工具:①教学模式(带读教领域,非摘要/问答)②备课(预先顺综述引文网络读承重论文、落带锚点笔记)③讲解锚定原文(点击跳回阅读器)④学习地图/进度点亮 ⑤面向进新领域的小白。

## 谁最近

NotebookLM 整体最近:唯一有真 tutor 模式(不直接给答案、追问带你、严格锚定源不跑题),带 mind map,学习定位。差四点:不会顺综述引文自己去读承重论文落笔记;它是笔记本不是阅读器,没有精排 PDF/划线/批注/圈选,做不了锚回坐标下钻;地图无进度点亮;不面向无教科书科研领域的小白。这四点恰好都是我们的地基(阅读器现成,备课是 v1 唯一新基建)。

分项最近:备课机制是 Undermind(顺引文读数百篇现搭领域模型,但为检索不为教学、笔记不可见);锚定阅读是 Semantic Reader(但不教不备课)。

## 空白点(差异化)

没有产品把三件事拧在一根综述脊柱上:tutor 沿综述带读教领域 + 后台预先消化引文网络落带页码锚点的备课笔记 + 全部锚回可划线批注圈选的专业阅读器下钻。且没搜到 2025-2026 新创业公司做这个组合(上线前再扫一次)。次一层空白:把备课做成可见的、随进度懒惰推进的管线——没人把预读产物摊给用户看。

## 值得偷的

- Semantic Reader:悬停引文就地弹卡片带 TLDR;引文按"读过没/在不在库里"上色——后者天然是"进度点亮"最便宜的第一形态,不用画图谱先给引文上色。https://www.semanticscholar.org/product/semantic-reader ,机制论文 https://cacm.acm.org/research/the-semantic-reader-project/
- PapersGPT AutoPilot:隔夜批量消化 + 笔记自动落库,备课管线的工程形态参考。https://github.com/papersgpt/papersgpt-for-zotero
- Undermind Timeline:历史发展/研究组/关键人物,比引文图更适合小白的地图呈现,留给后移的地图 UI。https://www.undermind.ai/
- Elicit:每条论断挂句级出处,锚定纪律的标杆。https://elicit.com/solutions/systematic-review
- Connected Papers/ResearchRabbit:单种子扩图、迭代下钻的交互模型,契合"引文网络是延伸阅读清单"。https://www.researchrabbit.ai/features
- NotebookLM:"对话风格开关"轻交互,验证了课堂模式按钮这条路。https://blog.google/innovation-and-ai/models-and-research/google-labs/notebooklm-student-features/

## 存疑

NotebookLM 是否能精确锚回上传 PDF 的页码坐标未验证——它给源引用,但做不了"点击跳阅读器原位精读"。
