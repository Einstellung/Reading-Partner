# 语音输入（M-voice-1）

## 设计

按住说话，做出 Wispr Flow 的两段式效果：录音 → STT 转写 → LLM 润色 → 成稿落进聊天输入框，用户看一眼再发。不自动发送。

聊天输入框旁加一个麦克风按钮。pointerdown 开始录，松手结束并触发转写+润色；录音中显示红点和秒数，转写中显示 spinner。指针按住后移出按钮再松手＝取消，Esc 也取消。没配 STT key 时点麦克风给一行提示指向设置。

录音在 Rust 侧（cpal + hound）。WebKitGTK 的 getUserMedia/MediaRecorder 在 Linux 上历史性地不可靠，而本项目主力桌面是 Linux。Rust 采集设备原生采样率，混成单声道、线性重采样到 16k、写 16-bit PCM WAV（SenseVoice/whisper 类模型要 16k 单声道）。cpal 的 Stream 不是 Send，所以用独立线程持有 stream，AtomicBool 通知停止，回调把采样推进共享 buffer；start/stop/cancel 三个命令驱动这个线程。录音上限 90s 兜底。

STT 请求走 Tauri http 插件（cleanTauriFetch），绕过 webview 的 CORS 和 CSP（connect-src 拦直连跨源，插件走 IPC）；任意 https STT 主机都能用。

## STT 可插拔层

OpenAI 兼容的转写客户端：POST `{base}/v1/audio/transcriptions`，multipart 带 file(audio.wav) + model，Bearer key，响应容忍 `{text}` 和纯字符串。默认硅基流动 `https://api.siliconflow.cn` + `FunAudioLLM/SenseVoiceSmall`（免费、中文最强、OpenAI 兼容）。base/model 存 settings.json（跟随同步），key 存 credentials.json（不同步，和图片中继 key 一个路子）。设置里可换任意 OpenAI 兼容供应商。

调研结论：小米 ASR 不免费；Groq 的 whisper 中文弱；本地离线（sherpa-onnx 等）后置。

## 润色 pass

一次调用用户现有的聊天供应商，低思考。系统提示：清理口语转写成可直接输入的文本——去填充词、自我更正只留改后版本、加标点断句、保持原语言（中文保持中文、英文保持英文、混合保持混合、不翻译）、按术语表纠正明显听错的专名，不加内容、不作答、只输出清理后的文本。术语表＝当前书标题 + 大纲/章节标题（取自 fulltext 缓存），约 300 字封顶。润色调用失败就回退原始转写——语音输入只降级不报废。

## 已知限制

- iPad 后置：录音是 desktop cpal 路径，iOS 要另走 AVAudioRecorder。已在 Info.plist 预置 NSMicrophoneUsageDescription。
- 无流式预览：转写是录完整段一次性出结果，录音中不显示实时文字。
- WAV 字节经 IPC 以数字数组回传，长录音偏重；典型几秒可忽略，90s 上限兜底。
