# 图片讲解：AI 拿着论文的图讲

2026-07-17 拍板。动机：论文的图承载大量信息，AI 讲解时只能引用图注文本，用户得自己翻回去看图。

## 形状

功能分两半，共享同一个 figure 索引：

- 输出侧：AI 回复里写 `[fig:3]`，聊天渲染成内嵌图片卡片（按边界框从渲染页裁剪），点击跳回原页。与 `[p.N]` 引用同一套语法体系。
- 输入侧：`view_figure` 工具，把裁剪图作为图片喂给视觉模型，AI 才能真的回答"图里那个箭头是什么"。备课消化时同样可用。

figure 索引在全文提取时顺带建：pdfjs 枚举每页图片对象取边界框，配对最近的 "Figure N:" 图注，得 `{编号, 页码, 图注, bbox}`，落盘在 fulltext 缓存旁。AI 上下文带图目录（编号+页码+图注，省 token）。

## 已知限制与兜底

- bbox+图注配对是启发式：双栏、多子图（3a/3b）、跨栏图会配错。配不准退化为整页缩略图。
- 扫描版 PDF 整页是一张图，天然整页。
- 图目录进上下文要设上限，超长综述截断到当前章节附近。
