# 账户同步

## 动机

跨设备接续：PC 上班时聊，回家 iPad 接着聊。数据一致性主张全量同步，包括 PDF 本身。

## 机制

Google 账户登录 + Google Drive 作用户自带存储，不搭后端。与"测试者自带 API key"同一哲学：数据和算力都在用户自己账下。

scope 用 `drive.file`，在用户 Drive 里建可见的 "Reading Partner" 文件夹。不用 `appDataFolder`：Google 文档明确说明用户断开应用授权时 `appDataFolder` 会被整个删除，把书和批注放进去会造成数据事故。可见文件夹用户能自行备份，断开授权也不丢数据。app 按 file id 追踪，用户重命名文件夹不影响。

## Drive 布局

```
Reading Partner/
  books/<contentHash>.pdf     内容寻址，不可变 blob，上传一次永不冲突
  data/                       各 JSON（reading-state、annotations、threads、topics、memory…）
  manifest.json               每文件的 rev / mtime
```

## 同步时机

- 本地写入 debounce 后 push。
- 启动时 + 周期数分钟拉 manifest，比对 rev，有新的下拉。
- 冲突 per-file last-writer-wins。单用户两设备，基本不并发。

## 不同步

- fulltext / figures 缓存：按内容 hash keyed，另一端自动重建。
- events 日志：按设备各自追加。
- prep 下载的论文 PDF（`prep-<hash>/pdf/`）及其派生缓存：另一端凭 plan 重新下载，避免 prep 目录几百 MB 上云。prep 只同步 `state.json` 和笔记 md。
- 缩略图 / 对话粘贴的图片（`images/`）：v1 不同步；另一端显示为缺图，`readThreadImages` 已容忍。
- sync 自身：`sync-auth.json`（token）、`sync-state.json`（folder/file id 映射、上次同步快照、自动同步开关）。

## 自动同步开关

开关值存 `sync-state.json`（不同步），不放 settings.json，从根上避免"关了同步却被同步回来打开"的怪圈。同步启用本就是每台设备各自的选择。

## AI 凭据同步

`credentials.json` 在同步范围内（含 Anthropic OAuth token）。两台设备共用一份 refresh token 有被服务端轮换后互相失效的风险；单用户两设备低频使用可接受，出问题重新登录即可。

## 设置页 Sync 区块

Google 登录按钮 → 登录后显示账户邮箱、自动同步开关（登录后默认开，可手动关）、Sync now、上次同步时间、登出。

## OAuth

- 桌面：PKCE + loopback，refresh token 落本地。复用 Anthropic 登录已有的 Rust 一次性回调监听器（`start_oauth_callback_listener`，`oauth_callback.rs`），不引 tauri-plugin-oauth——同一个固定 loopback 端口、按 state 匹配，少一套机制。client id/secret 从 `VITE_GOOGLE_CLIENT_ID` / `VITE_GOOGLE_CLIENT_SECRET` 读，Desktop client 的 secret 不保密（PKCE 才是保护）。
- iOS：custom URL scheme，需单独的 iOS OAuth client，等 TestFlight 通了再建。
- Google Console 需两个 client：Desktop + iOS。

坑：OAuth consent screen 处于 Testing 状态时 refresh token 七天过期，必须发布 In production。非敏感 scope 可直接发布，代价是"未验证应用"警告，品牌验证可去掉。

## 前置

所有数据身份从 `hashPath`（绝对路径 hash）换成文件内容 hash，配套本地 library 导入模型。这也正是 iPad sandbox 需要的形态。

## 里程碑

- M-sync-1：内容 hash + library（纯本地）。
- M-sync-2：OAuth + Drive backend + 同步引擎 + 设置 UI。
- iOS client 后置。

## 外部依赖

用户需建 Google Cloud 项目：consent screen 发布 In production、开 Drive API、建 Desktop OAuth client。
